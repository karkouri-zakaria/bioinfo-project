# Agent Specification: Single-Container Neuroimaging App

## 1. Project Overview
This project is an interactive web-based application designed to run entirely within a single containerized development environment. It takes structural brain MRI scans (NIfTI format: `.nii` or `.nii.gz`) from repositories like OpenNeuro, processes them using a local Python backend powered by MONAI and PyTorch for automated structural segmentation, and instantly renders the results as an interactive 3D volume overlay using NiiVue in a React + TypeScript + Tailwind CSS frontend.

---

## 2. Environment & Tech Stack

*   **Runtime Environment:** Current active development container (assumes Python 3.10+ and Node.js/npm are available globally).
*   **Backend Server:** FastAPI running on `http://127.0.0.1:8000` with Uvicorn.
*   **Machine Learning Engine:** PyTorch + MONAI (Medical Open Network for AI) + NiBabel.
*   **Frontend UI App:** React 18+ (scaffolded via Vite) + TypeScript + Tailwind CSS.
*   **3D Visualization Core:** NiiVue (`niivue` library mounted via standard WebGL canvas canvas).
*   **Network Routing:** Vite dev server running on port `5173` utilizing a proxy configuration to route all `/api/*` traffic transparently to port `8000`.

---

## 3. Strict Development Constraints & Guardrails

> **CRITICAL RULE 1: Affine Matrix Integrity**
> You must never drop, reset, or corrupt the structural orientation spatial data (the `affine` matrix) of the input NIfTI file. All preprocessing steps must utilize dictionary-based transforms from `monai.transforms` that preserve header metadata. The output segmentation mask generated by the model must be saved back into a NIfTI format using the exact spatial header and affine matrix of the original source volume. Failure to do this will result in misaligned 3D overlays in the browser canvas.

> **CRITICAL RULE 2: Decoupled WebGL Render Cycle**
> The initialization and lifecycle management of the `Niivue` instance must be strictly decoupled from React's standard element rendering loop. You must mount NiiVue onto a persistent HTML5 `<canvas>` using a React `useRef` hook inside a dedicated canvas wrapper component. Do not trigger global or local React state updates on every WebGL frame update to avoid memory exhaustion and garbage collection crashes inside the container.

> **CRITICAL RULE 3: No External Isolation Layers**
> Do not generate any new Dockerfiles, Docker Compose scripts, or isolated environment layer scripts. All pip packages and npm dependencies must be installed directly into the current active container environment.

## 5. Detailed Phase-by-Phase Build Order

### Phase 1: Local Backend Base Setup
Dependencies: Create backend/requirements.txt containing fastapi, uvicorn, nibabel, torch, and monai. Install them using pip install -r backend/requirements.txt.

API Gateway: Write backend/main.py. Configure explicit CORSMiddleware tracking allowed origins. Implement a single async POST /api/segment endpoint that accepts a multipart binary file stream via UploadFile.

Processing Logic: Draft backend/pipeline.py. Implement a baseline monai.transforms sequence: LoadImaged, EnsureChannelFirstd, and ScaleIntensityRanged. For initial validation, have the model inference block perform a basic thresholding operation or a dummy pass that duplicates the input volume matrix as an integer mask label. Save this label back to a temporary .nii.gz file copying the input volume's affine matrix via NiBabel, and serve it via FastAPI's FileResponse.

### Phase 2: Client Setup & Proxies
Scaffolding: In the root directory, scaffold the client application folder directly via Vite: npm create vite@latest frontend -- --template react-ts.

UI Tooling: Move into the frontend folder and install the required modules: npm install niivue tailwindcss postcss autoprefixer. Run npx tailwindcss init -p to build the configurations, and add the Tailwind template targets for your source files. Include @tailwind base;, @tailwind components;, and @tailwind utilities; in frontend/src/index.css.

Routing Proxy: Update frontend/vite.config.ts. Inside the configuration object, define a server.proxy object mapping /api traffic directly to target destination http://127.0.0.1:8000, ensuring developers do not have to worry about cross-origin hardcoding.

### Phase 3: Modular Front-End UI Construction
Canvas Wrapper: Build frontend/src/components/VolumeViewer.tsx. Set up an HTML5 <canvas id="niivue-canvas" ref={canvasRef}></canvas> element wrapped in a responsive Tailwind box. Use a React useEffect block containing a single-instantiation check to bind a new Niivue() configuration to the element ref, loading volume URLs dynamically whenever props update.

Dashboard Shell: Build frontend/src/components/Dashboard.tsx. Design a drag-and-drop file interface zone styled with Tailwind utility classes. Add file type validations restricting inputs to .nii and .nii.gz.

State Integration: Write the async upload handler. When an OpenNeuro file is uploaded, use standard fetch or axios to stream the payload to /api/segment. Catch the returned binary blob mask, convert it to an object URL, and push both the base scan URL and mask overlay URL directly into the VolumeViewer component array using nv.addVolumeByUrl().

### Phase 4: Pipeline Execution & Verification Testing
Execution Check: Start both servers concurrently using separate terminal tabs in your opened editor:

Terminal 1: cd backend && uvicorn main:app --reload --port 8000

Terminal 2: cd frontend && npm run dev
